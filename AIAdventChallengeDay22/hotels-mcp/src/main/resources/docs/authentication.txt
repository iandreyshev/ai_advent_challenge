# Аутентификация в TaskFlow

## Способы входа

TaskFlow поддерживает несколько способов аутентификации:

1. **Email и пароль** - стандартный способ входа
   - Минимальная длина пароля: 8 символов
   - Требуется подтверждение email при регистрации
   - Возможность восстановления пароля через email

2. **OAuth провайдеры**
   - Google OAuth 2.0
   - GitHub OAuth
   - Microsoft Azure AD

## Решение проблем с авторизацией

### Ошибка "Invalid redirect URI" при OAuth
Эта ошибка возникает когда:
- Вы используете неправильный домен (например, localhost вместо production URL)
- Redirect URI не зарегистрирован в настройках OAuth приложения
- Есть опечатка в redirect URL

**Решение:**
1. Убедитесь, что используете правильный домен
2. Проверьте настройки OAuth приложения в консоли провайдера
3. Очистите кэш браузера и cookies
4. Попробуйте использовать режим инкогнито

### Не приходит письмо подтверждения
- Проверьте папку "Спам"
- Убедитесь, что email введен правильно
- Письмо может прийти в течение 5-10 минут
- Используйте кнопку "Отправить повторно" в настройках аккаунта

### Забыл пароль
1. Нажмите "Забыли пароль?" на странице входа
2. Введите ваш email
3. Перейдите по ссылке из письма
4. Создайте новый пароль

### Аккаунт заблокирован
Аккаунт может быть временно заблокирован после 5 неудачных попыток входа.
Блокировка снимается автоматически через 30 минут или можно восстановить доступ через email.

## Безопасность

- Используйте двухфакторную аутентификацию (2FA) для дополнительной защиты
- Регулярно меняйте пароль
- Не используйте один и тот же пароль на разных сайтах
- Выходите из аккаунта на общих компьютерах
